< classes vs. ids >

When it comes to developing websites, we use a concept call CSS (Cascading Style Sheets). CSS allows us to add design and style properties to our website, and works in conjunction with our HTML doc. In CSS you can control the style of the html elements through selectors. ID and class are the most used CSS Selectors. In this blog post I will explain both ID & Class selectors and discuss their differences and when is best to use them.

ID


The ID selector is used to select a single HTML element. ID selectors can only be used on one HTML element and are unique in id attribute value.

CLASS


The Class selector can be used to select a single or group of multiple HTML elements, and used the same class attribute value.

The main difference between these two selectors, is that ID selectors can only be used for a single element while class selector can be used for both one element or a group of multiple elements. In an HTML doc there can only be one ID element with a certain attribute value and this is often used on an element that there is typically only one of e.g. footer, header and menu.

The basic rule to keep in mind when deciding which selector to use is that ID selectors are used for single elements that appear on a page only once, and class is used for single or multiple elements that appear on the page more than once.

Something to keep in mind is that a HTML element can have both an ID and Class. For example, let’s say you need to have two boxes on your page with the same size and style but different positioning. In this case, you can assign the same class to these boxes to control their size and style and then assign a different id to each box to control their positions.

In conclusion, ID’s and Classes are extremely important when it comes to adding style to your website and are important to be used correctly as it could result in a non-functional HTML code.
